# Анализ SlotsJs.cs

Файл `SlotsJs.cs` является частью `PostFilter` и предназначен для модификации JavaScript-файла `/js/slots.js`.

## Важное замечание

**Весь код внутри метода `SlotsJs` в оригинальном C#-проекте закомментирован.** Это означает, что фильтр был отключен и не применял никаких модификаций, а просто возвращал исходный файл как есть.

## Функциональность (если бы была активна)

Закомментированный код был предназначен для подмены стандартных образов персонажей на кастомные.

1.  **Перехват функции**: Модифицировалась функция `show_obraz`, отвечающая за отображение образа персонажа.
2.  **Вызов `ObrazSubstitute`**: Внедрялся вызов `window.external.ObrazSubstitute(nick, image)`, который позволял хост-приложению подменить путь к файлу образа (`image`) для конкретного персонажа (`nick`).
3.  **Рекламная плашка**: Если образ подменялся на кастомный, загруженный с домена `abclient.1gb.ru`, под ним добавлялась плашка с текстом "ABClient" и информацией о покупке индивидуального образа.
4.  **Замена `alt` на `title`**: Стандартное исправление для совместимости.

## План портирования в Java

Поскольку оригинальная функциональность отключена, при портировании мы сохраним это поведение.

1.  **Создать `SlotsJs.java`**: В пакете `ru.neverlands.abclient.postfilter`.
2.  **Реализовать метод `process`**:
    *   Метод будет принимать `byte[] array`.
    *   Он будет **просто возвращать `array` без каких-либо изменений**.
    *   В комментарии к методу будет указано, что оригинальная логика была отключена.
3.  **Интегрировать в `Filter.java`**:
    *   В `Filter.java` найти условие `address.endsWith("/js/slots.js")`.
    *   Вызывать из него `SlotsJs.process(array)`.
