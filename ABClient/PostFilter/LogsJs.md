# Анализ LogsJs.cs

Файл `LogsJs.cs` является частью `PostFilter` и предназначен для модификации JavaScript-файлов, относящихся к логам (например, `/js/logs_comp.js`).

## Важное замечание

**Основная логика внутри метода `LogsJs` в оригинальном C#-проекте закомментирована.** Как и в случае со `SlotsJs.cs`, это означает, что фильтр был отключен и не применял никаких модификаций.

## Функциональность (если бы была активна)

Закомментированный код был предназначен для перехвата и обработки системных сообщений из логов, в частности — результата разделки животных.

1.  **Удаление счетчиков**: Первая строка `sb.Replace(AppConstants.HtmlCounters, string.Empty);` должна была удалять из HTML-кода различные счетчики статистики (например, LiveInternet).
2.  **Перехват результата разделки**:
    *   **Контекст**: Модифицировался `switch` в JS-коде, который обрабатывает разные типы записей в логе.
    *   **Модификация**: В `case 8`, отвечающий за результат разделки, добавлялся вызов `window.external.ShowRazd(razd)`.
    *   **Назначение**: Это позволяло передать строку с результатом разделки (`razd`) в хост-приложение (C#) для дальнейшей обработки, например, для отображения в отдельном окне или сохранения в статистику.

## План портирования в Java

Поскольку оригинальная функциональность отключена, при портировании мы сохраним это поведение.

1.  **Создать `LogsJs.java`**: В пакете `ru.neverlands.abclient.postfilter`.
2.  **Реализовать метод `process`**:
    *   Метод будет принимать `byte[] array`.
    *   Он будет **просто возвращать `array` без каких-либо изменений**.
    *   В комментарии к методу будет указано, что оригинальная логика была отключена.
3.  **Интегрировать в `Filter.java`**:
    *   В `Filter.java` найти условие `address.contains("/js/logs")`.
    *   Вызывать из него `LogsJs.process(array)`.
