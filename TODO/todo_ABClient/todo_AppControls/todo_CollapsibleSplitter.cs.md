### 1. План портирования CollapsibleSplitter.cs

Файл `CollapsibleSplitter.cs` определяет сложный кастомный UI-контрол. Это не просто разделитель, а гибрид, который можно как перетаскивать для изменения размера панелей, так и нажимать для полного сворачивания/разворачивания одной из панелей.

### 2. Функциональность в C#

- **Наследование:** Класс наследуется от `System.Windows.Forms.Splitter`, что дает ему базовую функциональность перетаскивания для изменения размеров соседних панелей.
- **Двойное назначение:**
    1.  **Разделитель:** Вне центральной "горячей зоны", он ведет себя как стандартный сплиттер, меняя курсор на `VSplit` или `HSplit`.
    2.  **Кнопка:** В центре контрола есть "горячая зона" (`_rr`). При наведении на нее курсор меняется на `Hand`, а клик вызывает метод `ToggleState()`.
- **`ToggleState()`:** Этот метод изменяет свойство `Visible` у привязанного контрола (`ControlToHide`), эффективно сворачивая или разворачивая его. Опционально может изменять размер родительской формы (`_expandParentForm`).
- **Кастомная отрисовка (`OnPaint`):** Вся визуальная часть контрола рисуется вручную с помощью GDI+. Это включает в себя фон, 3D-границы, стрелки (указывающие направление сворачивания) и набор точек-захватов в различных стилях (`XP`, `Win9X`, `DoubleDots` и т.д.). Этот код полностью специфичен для `System.Drawing` и непереносим.

### 3. Решение для портирования на Android

Прямое портирование этого контрола невозможно. Необходимо воссоздать его функциональность, используя идиоматичные для Android компоненты. Основная цель — предоставить скрываемую боковую панель.

**Рекомендуемый подход: `DrawerLayout`**

Наиболее стандартным и узнаваемым для пользователя Android-паттерном для скрывающихся боковых панелей является `DrawerLayout` (выезжающее меню).

- **Преимущества:**
    - Стандартный компонент, привычный пользователям.
    - Обрабатывает анимации, жесты открытия/закрытия (свайп от края экрана).
    - Легко интегрируется с `Toolbar` и `ActionBar`.
- **Недостаток:** Не предоставляет возможности изменять размер панели перетаскиванием. Однако, эта функциональность менее важна на мобильных устройствах с ограниченным пространством экрана.

**Альтернативный подход (если изменение размера критично): Кастомный разделитель**

Если возможность изменять размер панели является обязательной, можно создать кастомный компонент.

- **Структура:** `LinearLayout` (горизонтальный) с тремя дочерними элементами: левая панель, `View`-разделитель, основная панель.
- **Изменение размера:** На `View`-разделитель вешается `OnTouchListener`. В обработчике `ACTION_MOVE` изменяются `layout_weight` левой и основной панелей.
- **Сворачивание:** Рядом с разделителем помещается `ImageButton`, который по клику меняет `visibility` левой панели на `View.GONE`.

**Вывод:** Рекомендуется использовать `DrawerLayout` как более простое, стандартное и подходящее для мобильных устройств решение. Кастомный разделитель следует реализовывать только в случае крайней необходимости.

### 4. План реализации (с использованием `DrawerLayout`)

1.  **Обновить `activity_main.xml`:**
    - [ ] Корневым элементом разметки сделать `androidx.drawerlayout.widget.DrawerLayout`.
    - [ ] Внутри него разместить два дочерних элемента:
        1.  Основной контент (например, `FrameLayout` или `ViewPager2` для вкладок).
        2.  Боковую панель — `com.google.android.material.navigation.NavigationView` с `android:layout_gravity="start"`.
2.  **Заполнить `NavigationView`:**
    - [ ] В `NavigationView` разместить контент, который ранее находился на сворачиваемой панели в C#-приложении (список контактов, таймеры и т.д.). Вероятно, это будет `RecyclerView` или `FragmentContainerView`.
3.  **Интегрировать с `Toolbar`:**
    - [ ] Создать `ActionBarDrawerToggle` для управления открытием/закрытием `DrawerLayout` с помощью стандартной иконки-"бургера" в `Toolbar`.
4.  **Отказаться от `CollapsibleSplitter`:**
    - [x] Логика самого контрола не портируется. Его функциональность заменяется стандартными компонентами Android.
