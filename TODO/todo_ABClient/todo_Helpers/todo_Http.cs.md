# План портирования Http.cs

Файл `Http.cs` - это статический класс с низкоуровневыми вспомогательными методами для работы с HTTP.

## Функциональность в C#

*   **Назначение**: Предоставить утилиты для ручной обработки различных аспектов HTTP-протокола. Вероятно, использовался в рамках кастомного прокси-сервера.
*   **Ключевые методы**:
    *   `IPEndPointFromHostPortString`: Парсит строку с хостом и портом.
    *   `DoUnchunk`: Декодирует тело ответа, использующее `Transfer-Encoding: chunked`.
    *   `GZipExpand` / `DeflateExpand`: Распаковывает сжатые GZip и Deflate данные.

## Проверка на существующую реализацию в Android

- **Результат:** Функциональность не портирована, но в ней нет необходимости.
- **Объяснение**:
    1.  В Android-проекте в качестве HTTP-клиента используется библиотека `OkHttp`.
    2.  Современные HTTP-клиенты, включая `OkHttp`, обрабатывают все эти низкоуровневые детали протокола **автоматически и прозрачно**.
    3.  `OkHttp` самостоятельно выполняет DNS-запросы, декодирует `chunked` ответы и распаковывает `gzip` и `deflate` сжатие без необходимости какого-либо ручного вмешательства.

## Решение для портирования на Android

Никаких действий не требуется. Весь функционал этого класса предоставляется "из коробки" используемой в проекте библиотекой `OkHttp`.
