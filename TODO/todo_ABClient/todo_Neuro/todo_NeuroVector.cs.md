
### 1. План портирования NeuroVector.cs

Файл `NeuroVector.cs` является классом данных для `NeuroBase`. Он представляет собой один "обученный" вектор, соответствующий определенному символу.

### 2. Функциональность в C#

- **Назначение:** Хранить один эталонный вектор (100-мерный) и соответствующий ему символ (`char`).
- **Данные:**
    - `token`: Символ (буква или цифра), который представляет этот вектор.
    - `vector`: Массив из 100 `double`, представляющий нормализованное изображение символа 10x10 пикселей.
- **Сериализация:** Имеет методы для сохранения и загрузки из `BinaryReader`/`BinaryWriter`. Используется для работы с файлом `abneuro.dat`.
- **`Distance()`:** Вычисляет квадрат евклидова расстояния до другого вектора. Это основная операция для алгоритма k-ближайших соседей, используемого в `NeuroBase`.

### 3. Решение для портирования на Android

Поскольку весь самописный OCR-движок (`NeuroBase`) заменяется на **Google ML Kit Text Recognition**, этот класс, являющийся его неотъемлемой частью, также **не требует портирования**.

Логика хранения эталонных образцов и сравнения с ними становится ненужной при использовании готовой, более совершенной библиотеки.

### 4. План реализации
