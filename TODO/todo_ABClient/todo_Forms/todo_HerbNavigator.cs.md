# План портирования HerbNavigator.cs

Файл `HerbNavigator.cs` реализует сложную форму для поиска трав и построения маршрута к ним.

## Функциональность в C#

*   **Назначение**: Предоставить пользователю инструмент для поиска расположения определенных трав на карте и автоматической навигации к ним.
*   **UI**:
    *   `checkedListSearch`: Список с чекбоксами, позволяющий выбрать одну или несколько трав для поиска.
    *   `button1` ("Поиск трав >>"): Кнопка для запуска процесса поиска.
    *   `listResult`: Список для отображения найденных локаций, где произрастают выбранные травы.
    *   `browserReport`: Встроенный браузер для отображения построенного пути к выбранной локации.
    *   `buttonOk` ("В путь!"): Кнопка для запуска автоматического движения по построенному маршруту.
    *   `checkBox1` ("Голос Мойры (VIP)"): VIP-опция, влияющая на поиск.
*   **Логика**: 
    1.  Пользователь отмечает интересующие его травы в `checkedListSearch`.
    2.  Нажимает кнопку поиска.
    3.  Приложение (логика не видна в `cs` или `designer.cs`, скорее всего в обработчиках событий) выполняет поиск по своим данным и выводит результат в `listResult`.
    4.  Пользователь выбирает одну из найденных локаций.
    5.  Приложение строит маршрут и отображает его в `browserReport`.
    6.  Кнопка "В путь!" становится активной, и нажатие на нее запускает навигацию.

## Проверка на существующую реализацию в Android

- **Результат:** Функциональность не реализована. В проекте есть только настройка `AutoHerb`, которая, вероятно, включает или выключает какой-то общий механизм авто-сбора, но отсутствует сложный UI для поиска и навигации.

## Решение для портирования на Android

Для портирования потребуется создать новую `Activity` или `Fragment`, который будет содержать все необходимые элементы управления. Логика поиска и построения маршрута потребует переноса соответствующих данных и алгоритмов.
