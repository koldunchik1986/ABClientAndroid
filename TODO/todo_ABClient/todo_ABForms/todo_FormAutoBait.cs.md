
### 1. План портирования FormAutoBait.cs

Файл `FormAutoBait.cs` представляет собой простую диалоговую форму для запуска функции "Автоприманка ботов".

### 2. Функциональность в C#

- **Назначение:** Получить от пользователя число — количество использований приманки.
- **UI:** Содержит текстовое поле для ввода числа и кнопку "Запустить приманки!".
- **Логика:** Код самого класса пуст. Логика подразумевается следующая: форма открывается модально (`ShowDialog()`), после нажатия на кнопку "Запустить" она закрывается с `DialogResult.OK`, и вызвавший ее код считывает значение из текстового поля.

### 3. Решение для портирования на Android

Такой простой диалог для ввода данных идеально реализуется с помощью `AlertDialog` с кастомным макетом.

- **UI:** Вместо отдельной формы будет использоваться `AlertDialog`.
- **Ввод данных:** В макет диалога будет помещен `EditText` для ввода числа.
- **Обработка результата:** Результат (введенное число) будет передаваться обратно в вызвавший код через лямбда-функцию (callback).

### 4. План реализации

1.  **Создать макет `dialog_auto_bait.xml`:**
    - Создать XML-файл в `res/layout`.
    - Внутри разместить `TextView` с текстом "Введите количество использований" и `EditText` с `inputType="number"`.
2.  **Создать утилитарную функцию:**
    - В подходящем утилитном классе (например, `DialogUtils.kt`) создать функцию `fun showAutoBaitDialog(context: Context, callback: (numberOfUses: Int) -> Unit)`.
    - Внутри этой функции:
        - Создать `AlertDialog.Builder`.
        - Загрузить кастомный макет `dialog_auto_bait.xml` с помощью `LayoutInflater`.
        - Установить этот макет как `view` для диалога.
        - Настроить кнопку "Запустить": в ее слушателе получить текст из `EditText`, преобразовать в `Int` и вызвать `callback` с этим числом.
        - Настроить кнопку "Отмена".
        - Показать диалог.
3.  **Интеграция:**
    - Код, который должен инициировать автоприманку (например, по нажатию на кнопку в `MainActivity`), будет вызывать `showAutoBaitDialog`.
    - Логика самой автоприманки будет запущена внутри лямбды-колбэка и должна выполняться в фоновом потоке (например, в корутине, запущенной из `ViewModel`).
