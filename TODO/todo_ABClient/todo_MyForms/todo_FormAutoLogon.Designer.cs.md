
### 1. План портирования FormAutoLogon.Designer.cs

Файл `FormAutoLogon.Designer.cs` содержит код, автоматически сгенерированный дизайнером Windows Forms. Он описывает внешний вид и расположение элементов управления на форме автоматического входа.

### 2. Функциональность в C#

- **Назначение:** Инициализировать и настроить все UI-компоненты формы `FormAutoLogon`.
- **Содержимое:**
    - `Label` для отображения имени пользователя.
    - `Label` с текстом "входит в игру".
    - `Button` "OK" (которая используется как кнопка с обратным отсчетом).
    - `Button` "Отмена".
    - `Timer` (`timerCountDown`) с интервалом в 1 секунду для обратного отсчета.

### 3. Решение для портирования на Android

Этот файл **не портируется напрямую**. Его содержимое служит **визуальным руководством** для создания аналогичного диалогового окна с помощью XML-разметки в Android.

### 4. План реализации

1.  **Проанализировать UI:** Изучить `InitializeComponent()` для понимания, какие элементы (`Label`, `Button`) присутствуют на форме.
2.  **Создать XML-разметку:**
    - Создать файл `layout/dialog_auto_logon.xml`.
    - В нем, используя `TextView` и `Button`, воссоздать внешний вид формы.
    - Текстовые метки ("входит в игру", "Отмена") следует вынести в `res/values/strings.xml`.
3.  **Связать с логикой:**
    - Логика, описанная в `FormAutoLogon.cs`, будет использовать эту разметку. Этот процесс уже описан в файле `todo_FormAutoLogon.cs.md`.

- [ ] Перенести статичный текст в `strings.xml`.
