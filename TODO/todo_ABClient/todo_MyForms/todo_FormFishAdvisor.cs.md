
### 1. План портирования FormFishAdvisor.cs

Файл `FormFishAdvisor.cs` реализует окно "Советник рыбака", которое помогает игроку выбрать оптимальное место для рыбалки.

### 2. Функциональность в C#

- **UI:** Окно с полем для ввода умения рыбалки, выпадающим списком для выбора максимального уровня ботов и таблицей для вывода результатов.
- **Логика:**
    - В коде жестко задан большой список (`List<FishTip>`) всех возможных мест для рыбалки. Каждый элемент содержит информацию о прибыльности, требуемом умении, локации, ботах и рекомендуемой наживке.
    - При нажатии на кнопку "Рассчитать":
        1.  Считывается умение рыбалки и выбранный максимальный уровень ботов.
        2.  Основной список фильтруется: остаются только те места, где умение игрока достаточно, а уровень ботов не превышает выбранный.
        3.  Отфильтрованный список сортируется (вероятно, по прибыльности).
        4.  Результаты выводятся в `ListView`.

### 3. Решение для портирования на Android

Эту функциональность следует реализовать как отдельное `Activity` или `Fragment`. Основное внимание следует уделить управлению данными.

- **Данные:** Жестко кодировать большой список в коде - плохая практика. Эти данные следует вынести.
    - **Вариант А (предпочтительный):** Создать таблицу в базе данных Room для хранения всех "советов" и загружать их оттуда.
    - **Вариант Б:** Поместить данные в JSON-файл в папку `assets` и парсить его при первом открытии экрана.
- **UI:** Использовать современные компоненты Android Jetpack.
    - `Spinner` вместо `ComboBox`.
    - `RecyclerView` вместо `ListView` для эффективного отображения списка.

### 4. План реализации

1.  **Создать модель данных:**
    - Создать data-класс `FishTip` в Kotlin.
    - Если используется Room, пометить его как `@Entity`.
2.  **Вынести данные:**
    - Создать `fish_tips.json` в `assets` и заполнить его данными из C#-кода.
    - Создать `FishTipRepository`, который будет отвечать за загрузку и парсинг этого JSON-файла (или за доступ к БД Room).
3.  **Создать UI:**
    - Создать `layout/activity_fish_advisor.xml`.
    - Разместить `EditText` для умения, `Spinner` для уровня ботов, `Button` для расчета и `RecyclerView` для результатов.
4.  **Реализовать логику:**
    - Создать `FishAdvisorViewModel`.
    - При нажатии на кнопку, `Activity` передает параметры (умение, уровень) в `ViewModel`.
    - `ViewModel` запрашивает данные у `FishTipRepository`, выполняет фильтрацию и сортировку.
    - Результат помещается в `LiveData<List<FishTip>>`.
5.  **Отобразить результат:**
    - `Activity` подписывается на `LiveData`.
    - При обновлении данных, они передаются в `RecyclerView.Adapter`, который отображает список советов.
