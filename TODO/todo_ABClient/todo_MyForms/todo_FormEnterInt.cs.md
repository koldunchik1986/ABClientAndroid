
### 1. План портирования FormEnterInt.cs

Файл `FormEnterInt.cs` реализует универсальное диалоговое окно для ввода целочисленного значения в заданном диапазоне.

### 2. Функциональность в C#

- **UI:** Диалоговое окно с заголовком, текстовым полем для ввода и меткой, описывающей диапазон (например, "Введите значение от 1 до 100").
- **Логика:**
    - В конструктор передаются заголовок, значение по умолчанию, минимальное и максимальное допустимые значения.
    - **Валидация:** Используется событие `Validating` у `TextBox`. При попытке уйти с поля ввода, запускается проверка.
    - Метод `ValidInt` проверяет, что значение не пустое, является целым числом и находится в заданном диапазоне.
    - Если валидация не проходит, компонент `ErrorProvider` показывает сообщение об ошибке рядом с полем ввода, и фокус остается на этом поле.

### 3. Решение для портирования на Android

Это стандартный диалог для ввода данных. Его следует реализовать с помощью `DialogFragment` и использовать `TextInputLayout` для отображения ошибок.

### 4. План реализации

1.  **Создать XML-разметку:**
    - Создать `layout/dialog_enter_integer.xml`.
    - Использовать `com.google.android.material.textfield.TextInputLayout` и вложенный в него `TextInputEditText`.
    - Установить для `TextInputEditText` `android:inputType="number"`.
2.  **Создать класс `EnterIntDialogFragment`:**
    - Унаследовать от `DialogFragment`.
    - Передавать в него `title`, `min`, `max` и `default_value` через `Bundle`.
3.  **Реализовать логику:**
    - В `onCreateDialog` или `onViewCreated` настроить диалог.
    - Не использовать `TextWatcher` для постоянной проверки. Вместо этого, валидацию проводить при нажатии на кнопку "OK".
    - В слушателе `OnClickListener` для кнопки "OK":
        - Прочитать текст из `EditText`.
        - Провести все проверки (на пустоту, на число, на диапазон).
        - Если проверка не пройдена, вызвать `textInputLayout.setError("Сообщение об ошибке")`.
        - Если проверка пройдена, очистить ошибку (`textInputLayout.setError(null)`), вернуть результат и закрыть диалог.
4.  **Возврат результата:**
    - Использовать `setFragmentResultListener` для асинхронного получения введенного числа в вызывающем компоненте.
