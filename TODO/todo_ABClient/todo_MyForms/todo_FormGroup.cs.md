
### 1. План портирования FormGroup.cs

Файл `FormGroup.cs` реализует простой диалог для ввода названия группы.

### 2. Функциональность в C#

- **UI:** Диалоговое окно с одним текстовым полем для ввода и кнопками OK/Отмена.
- **Логика:**
    - В конструктор можно передать начальное значение для поля ввода.
    - Кнопка "OK" активна только тогда, когда поле ввода не пустое (после удаления пробелов по краям).

### 3. Решение для портирования на Android

Это стандартный диалог для ввода текста. Его следует реализовать с помощью `DialogFragment`.

### 4. План реализации

1.  **Создать XML-разметку:**
    - Создать `layout/dialog_enter_text.xml` (можно сделать его универсальным и для других подобных диалогов).
    - Использовать `com.google.android.material.textfield.TextInputLayout` и вложенный в него `TextInputEditText`.
2.  **Создать класс `EnterTextDialogFragment`:**
    - Унаследовать от `DialogFragment`.
    - Передавать в него заголовок и значение по умолчанию через `Bundle`.
3.  **Реализовать логику:**
    - В слушателе `OnClickListener` для кнопки "OK":
        - Прочитать и обрезать (`trim()`) текст из `EditText`.
        - Если строка пустая, вызвать `textInputLayout.setError("Название не может быть пустым")`.
        - Если проверка пройдена, вернуть результат и закрыть диалог.
4.  **Возврат результата:**
    - Использовать `setFragmentResultListener` для асинхронного получения введенного текста в вызывающем компоненте.
