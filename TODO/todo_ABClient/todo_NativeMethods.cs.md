# План портирования NativeMethods.cs

Файл `NativeMethods.cs` - это статический класс, содержащий объявления P/Invoke для вызова низкоуровневых функций Windows API.

## Функциональность в C#

*   **Назначение**: Предоставить C#-коду доступ к не-управляемым системным библиотекам Windows (`wininet.dll`, `user32.dll`, `ole32.dll`).
*   **Используемые функции**:
    *   **wininet.dll**: Функции для управления кэшем Internet Explorer и установки опций WinINet (например, прокси).
    *   **user32.dll**: Функции для манипуляции окнами (`FlashWindow`, `SendMessage`).
    *   **ole32.dll**: Функция для отрисовки OLE-объектов.
    *   Также содержит свойства для определения версии Windows (`IsWinXP`, `IsWinVista`).

## Проверка на существующую реализацию в Android

- **Результат:** Функциональность не портируема, но решаемые ею задачи уже реализованы (или должны быть реализованы) стандартными средствами Android.
- **Объяснение**:
    1.  Весь класс на 100% зависит от операционной системы Windows и ее библиотек. Прямое портирование невозможно.
    2.  **Управление кэшем**: В Android решается через `WebView.clearCache()` и кастомную логику в `DataManager` / `Cache`.
    3.  **Настройка прокси**: В Android настраивается на уровне HTTP-клиента (например, `OkHttpClient.Builder().proxy(...)`).
    4.  **Манипуляции с UI**: В Android решаются через стандартные методы View и Activity (например, отправка уведомлений `Notification` вместо `FlashWindow`).
    5.  **Проверка версии ОС**: В Android выполняется через `android.os.Build.VERSION`.

## Решение для портирования на Android

Никаких действий не требуется. Этот класс не нужно и невозможно портировать. Его функциональность заменяется идиоматичными для Android API.
