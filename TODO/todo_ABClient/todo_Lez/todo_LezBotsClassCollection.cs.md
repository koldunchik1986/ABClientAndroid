
### 1. План портирования LezBotsClassCollection.cs - [+] Реализован в LezBotsClassCollection.java

Файл `LezBotsClassCollection.cs` представляет собой статическую коллекцию, которая хранит и управляет списком всех известных "классов" ботов.

### 2. Функциональность в C#

- **Назначение:** Служить единым источником данных о типах ботов.
- **Реализация:**
    - Это `static` класс, работающий как синглтон.
    - Данные о ботах (Орки, Гоблины и т.д.) **жестко закодированы** в статическом конструкторе.
    - Использует `SortedDictionary` для хранения и быстрого доступа к данным по `Id`.
    - Предоставляет методы для получения одного класса по `Id` и всего списка для UI (например, для `ComboBox`).

### 3. Решение для портирования на Android

Жесткое кодирование данных в коде — негибкая практика. При портировании эти данные следует вынести во внешний ресурс. Вместо статического класса лучше использовать паттерн "Репозиторий".

- **Источник данных:** Перенести все жестко закодированные данные в JSON-файл, который будет храниться в папке `src/main/assets/`.
- **Архитектура:** Создать синглтон-репозиторий (`object` в Kotlin), который будет отвечать за загрузку и предоставление этих данных.

### 4. План реализации

1.  **Создать `lez_bot_classes.json`:**
    - В папке `src/main/assets/` создать JSON-файл со списком всех классов ботов.
      ```json
      [
        { "id": 1, "name": "Все", "plural": "Все" },
        { "id": 10, "name": "Человек", "plural": "Люди" },
        { "id": 101, "name": "Орк", "plural": "Орки" },
        // ... и так далее
      ]
      ```
2.  **Создать `LezBotsClassRepository`:**
    - Создать `object LezBotsClassRepository` в Kotlin.
    - Внутри него будет `private val botClasses = mutableMapOf<Int, LezBotsClass>()`.
    - Реализовать метод `load(context: Context)`, который будет считывать JSON из `assets` (с помощью `Gson` или `Moshi`) и заполнять `botClasses`.
    - Этот метод должен вызываться один раз при старте приложения (например, в `Application.onCreate()`).
    - Реализовать публичные методы `getBotClass(id: Int)` и `getAllBotClasses()`, которые будут возвращать данные из `botClasses`.
