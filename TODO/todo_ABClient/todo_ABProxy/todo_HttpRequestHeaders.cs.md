
### 1. План портирования HttpRequestHeaders.cs

Файл `HttpRequestHeaders.cs` представляет собой класс для хранения и управления заголовками HTTP-запроса, наследуясь от `HttpHeaders`.

### 2. Функциональность в C#

- **Назначение:** Представлять полный набор заголовков HTTP-запроса, включая первую строку (метод, путь, версия).
- **Логика:**
    - Наследует всю функциональность `HttpHeaders`.
    - Добавляет свойства `HttpMethod` и `RequestPath` для хранения метода и пути запроса.
    - Хранит путь (`RequestPath`) одновременно в виде строки и массива байт (`RawPath`) для корректной обработки кодировок.
    - Переопределяет `ToString()` и `ToByteArray()` для полной сериализации объекта обратно в текстовый или байтовый вид HTTP-запроса. Эти методы могут конструировать первую строку запроса (например, `GET /path HTTP/1.1`) и добавлять к ней все заголовки.

### 3. Решение для портирования на Android

Как и в случае с `HttpHeaders`, этот класс является частью низкоуровневой реализации HTTP-парсера. При использовании современных подходов его портирование не требуется.

- **`WebViewClient.shouldInterceptRequest`**: Этот метод предоставляет объект `WebResourceRequest`, который уже содержит всю необходимую информацию: метод (`getMethod()`), URI (`getUrl()`), заголовки (`getRequestHeaders()`).
- **Библиотеки для прокси/HTTP:** Готовые библиотеки (например, `OkHttp`) предоставляют свои классы для представления запросов (например, `okhttp3.Request`), которые инкапсулируют метод, URL, заголовки и тело запроса.

### 4. План реализации

1.  **Не портировать класс:** Класс `HttpRequestHeaders` не следует портировать.
2.  **Использовать нативные/библиотечные объекты:**
    - При перехвате запроса от `WebView` использовать объект `WebResourceRequest`.
    - При создании собственных запросов (например, с помощью `OkHttp`) использовать `okhttp3.Request.Builder` для конструирования запроса и объект `okhttp3.Request` для его представления.

- [ ] Удалить `HttpRequestHeaders` из плана портирования.
- [ ] Использовать `WebResourceRequest` или `okhttp3.Request` для представления HTTP-запросов.
