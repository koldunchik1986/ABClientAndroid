
### 1. План портирования HttpHeaderItem.cs

Файл `HttpHeaderItem.cs` представляет собой простую структуру данных для хранения одного HTTP-заголовка (пары имя-значение).

### 2. Функциональность в C#

- **Назначение:** Хранить имя и значение одного HTTP-заголовка.
- **Логика:**
    - Содержит два публичных строковых свойства: `Name` и `Value`.
    - Реализует интерфейс `ICloneable` для создания поверхностной копии объекта.
    - Метод `ToString()` переопределен для вывода заголовка в стандартном формате `"Name: Value"`.

### 3. Решение для портирования на Android

Это тривиальный класс. Его можно легко заменить.

### 4. План реализации

- **Kotlin:** Создать `data class HttpHeaderItem(val name: String, val value: String)`.
- **Java:** Создать простой POJO-класс `HttpHeaderItem` с двумя полями и конструктором.
- **Альтернатива:** Вообще не создавать этот класс, а использовать `Pair<String, String>` или аналогичную структуру, если не требуется дополнительная логика.
