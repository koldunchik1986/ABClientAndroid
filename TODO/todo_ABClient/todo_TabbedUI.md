# План реализации горизонтальных вкладок (Tabbed UI)

## Задача
Добавить в верхнюю часть приложения (над HTML-фреймом) горизонтальные вкладки, аналогичные C#-версии (режим "горизонтальных вкладок").

## Требования
- Первая вкладка — "Основная" (оставить текущий функционал).
- Новые вкладки создаются динамически при открытии:
    - Профиля игрока (neverlands.ru/pinfo.cgi?Nickname)
    - Внешних ссылок (например, forum.neverlands.ru и др.)
- Пользователь может закрывать дополнительные вкладки.
- При закрытии вкладки активируется предыдущая.
- Название вкладки — по содержимому (ник, название сайта и т.д.).
- Сохранять преемственность с C# по архитектуре и именованию.

## Архитектурные решения
- Использовать TabLayout (или аналог) поверх WebView.
- Для каждой вкладки — отдельный WebView или переиспользование одного с разными URL.
- Хранить список открытых вкладок в памяти (структура TabInfo: title, url, isMain).
- Кнопка закрытия (крестик) на каждой вкладке, кроме основной.
