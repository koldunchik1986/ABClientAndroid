
### 1. План портирования MainPhpWear.cs

Файл `MainPhpWear.cs` содержит набор функций, отвечающих за автоматическое надевание/проверку предметов, в основном для автоматизации рыбалки и разделки.

### 2. Функциональность в C#

- **`MainPhpWearUd(html)`**: Основная логика для автоматического надевания удочек.
    - Использует `ParsedDressed` для определения надетых вещей.
    - Проверяет, надеты ли удочки, указанные в профиле (`FishHandOne`, `FishHandTwo`).
    - Если удочка не надета, ищет ее в инвентаре (используя `GetInvList`) и генерирует редирект для ее надевания.
    - Если слот занят, сначала генерирует редирект для снятия предмета из этого слота.
    - Если нужная удочка не найдена, отключает авто-рыбалку.

- **`GetInvList(html)`**: Вспомогательная функция, которая парсит инвентарь и возвращает список объектов `InvEntry`.

- **`MainPhpIsMustWearUd(html)`**: Проверяет, должен ли пользователь надеть удочку согласно настройкам.

- **`MainPhpWearKnife(html)`** и **`MainPhpArmedKinfe(html)`**: Логика для автоматического надевания разделочного ножа. Одна функция проверяет, надет ли нож, другая - находит и надевает его из инвентаря.

- **`MainPhpGetSkinRes(html)`**: Парсит страницу с результатами разделки, извлекает полученные ресурсы, сравнивает с предыдущими значениями и выводит отчет в чат.

### 3. Решение для портирования на Android

Это важная часть движка автоматизации, которая должна быть полностью реализована в логике Android-приложения.

### 4. План реализации

1.  **Создать класс `EquipmentManager` в Android:**
    - Будет отвечать за проверку и смену экипировки для различных задач (рыбалка, разделка).
2.  **Создать класс `DressedStateParser`:**
    - Аналог `ParsedDressed`. Будет парсить HTML и предоставлять информацию о том, что в данный момент надето на персонаже.
3.  **Интеграция с `AutoFishManager` и `AutoSkinManager`:**
    - Перед началом рыбалки `AutoFishManager` должен обратиться к `EquipmentManager`.
    - `EquipmentManager` проверит, надеты ли удочки. Если нет, он должен инициировать серию `WebView`-запросов для их надевания (сначала снять мешающие предметы, затем надеть нужные).
    - Аналогичная логика должна быть для `AutoSkinManager` и ножа.
4.  **Парсинг ресурсов:**
    - Логику из `MainPhpGetSkinRes` следует перенести в `AutoSkinManager` для анализа результатов разделки.
