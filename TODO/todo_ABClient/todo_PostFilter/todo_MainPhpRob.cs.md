
### 1. План портирования MainPhpRob.cs

Файл `MainPhpRob.cs` содержит логику для автоматического ограбления игрока после боя.

### 2. Функциональность в C#

- **Назначение:** Автоматически обокрасть игрока после победы в бою.
- **Логика:**
    1.  Ищет в HTML-коде страницы javascript-переменную `fight_ty`.
    2.  Парсит этот javascript-массив.
    3.  Проверяет 11-й элемент массива (`fight_ty[10]`). Если он существует и содержит данные, это означает, что доступно ограбление.
    4.  Извлекает из `fight_ty[10]` все необходимые параметры для создания ссылки на ограбление (`p`, `uid`, `vcode`).
    5.  Формирует URL для ограбления и генерирует HTML-код с редиректом на этот URL.

### 3. Решение для портирования на Android

Это логика автоматизации, которая должна быть реализована на стороне Android-приложения.

### 4. План реализации

1.  **Дополнить `FightEngine` или `PostFightManager`:**
    - После завершения боя, необходимо анализировать страницу с результатами.
2.  **Реализовать парсинг `fight_ty`:**
    - Создать метод, который будет извлекать и парсить массив `fight_ty` из HTML.
3.  **Принятие решения и выполнение:**
    - Если парсер обнаруживает возможность ограбления, менеджер должен сформировать и выполнить GET-запрос на URL ограбления.

- [ ] Дополнить менеджер, отвечающий за действия после боя.
- [ ] Реализовать в нем парсинг переменной `fight_ty`.
- [ ] Реализовать формирование и выполнение запроса на ограбление.
