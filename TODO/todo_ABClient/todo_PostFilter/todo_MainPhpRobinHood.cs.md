
### 1. План портирования MainPhpRobinHood.cs

Файл `MainPhpRobinHood.cs` содержит очень необычную логику, которая, по-видимому, является своего рода анти-кражной системой, случайным образом передающей половину денег пользователя другому игроку или NPC.

### 2. Функциональность в C#

- **Назначение:** Периодически и случайным образом передавать половину NV и DNV пользователя другим персонажам.
- **Логика:**
    1.  **Парсинг денег:** Извлекает из HTML текущее количество NV и DNV пользователя.
    2.  **Передача NV:**
        *   Рассчитывает половину от текущего NV.
        *   С вероятностью 10% (`Dice.Make(10) == 0`), если эта сумма больше 1 NV и на странице нет подтверждения недавней передачи денег, запускается процесс передачи.
        *   Выбирается случайный игрок из текущей комнаты.
        *   Парсятся параметры из javascript-функции `transferform`.
        *   Генерируется и автоматически отправляется форма для передачи половины NV этому случайному игроку с комментарием "противоугонка ABClient".
    3.  **Передача DNV:**
        *   Аналогично, с 10% вероятностью передает половину DNV (если сумма больше 1) персонажу "Шандор-Волшебник".

### 3. Решение для портирования на Android

Назначение этой функции сомнительно и потенциально вредно для пользователя. **Настоятельно рекомендуется не портировать эту функциональность.**

Если же портирование все-таки необходимо, логика должна быть реализована на стороне Android-приложения.

### 4. План реализации (если необходимо)

1.  **Создать класс `RobinHoodManager`:**
    - Будет инкапсулировать эту логику.
2.  **Интеграция с `WebViewClient`:**
    - После загрузки `main.php` передавать HTML в `RobinHoodManager`.
3.  **Реализация логики в `RobinHoodManager`:**
    - Парсить NV и DNV.
    - Реализовать вероятностный запуск.
    - Получать список игроков в комнате.
    - Парсить параметры форм `transferform` и `m_form`.
    - Формировать и выполнять `POST`-запросы для передачи денег.

- [ ] **(НЕ РЕКОМЕНДУЕТСЯ)** Создать класс `RobinHoodManager`.
- [ ] **(НЕ РЕКОМЕНДУЕТСЯ)** Реализовать всю логику парсинга, принятия решений и выполнения `POST`-запросов.
