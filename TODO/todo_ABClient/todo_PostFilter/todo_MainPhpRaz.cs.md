
### 1. План портирования MainPhpRaz.cs

Файл `MainPhpRaz.cs` содержит логику для автоматической разделки монстра после боя.

### 2. Функциональность в C#

- **Назначение:** Автоматически разделать монстра для получения ресурсов.
- **Логика:**
    1.  Ищет в HTML-коде страницы javascript-переменную `fight_ty`.
    2.  Парсит этот javascript-массив.
    3.  Проверяет 10-й элемент массива (`fight_ty[9]`). Если он существует и содержит данные, это означает, что доступна разделка.
    4.  Извлекает из `fight_ty[9]` все необходимые параметры для создания ссылки на разделку (`type`, `p`, `uid`, `s`, `m`, `vcode`).
    5.  Формирует URL для разделки и генерирует HTML-код с редиректом на этот URL.

### 3. Решение для портирования на Android

Это логика автоматизации, которая должна быть реализована на стороне Android-приложения.

### 4. План реализации

1.  **Дополнить `FightEngine` или `PostFightManager`:**
    - После завершения боя, необходимо анализировать страницу с результатами.
2.  **Реализовать парсинг `fight_ty`:**
    - Создать метод, который будет извлекать и парсить массив `fight_ty` из HTML.
3.  **Принятие решения и выполнение:**
    - Если парсер обнаруживает возможность разделки, менеджер должен сформировать и выполнить GET-запрос на URL разделки.
