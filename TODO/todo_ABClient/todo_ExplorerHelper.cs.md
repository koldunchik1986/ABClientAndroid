# План портирования ExplorerHelper.cs

Файл `ExplorerHelper.cs` - это статический класс для программной очистки кэша Internet Explorer.

## Функциональность в C#

*   **Назначение**: Предоставить приложению возможность принудительно очистить кэш встроенного компонента `WebBrowser`, который в C# `WinForms` является оберткой над Internet Explorer.
*   **Реализация**: Очень низкоуровневая и специфичная для Windows. Использует `P/Invoke` для вызова функций из системной библиотеки `wininet.dll` (`FindFirstUrlCacheGroup`, `DeleteUrlCacheEntry` и т.д.) для перечисления и удаления записей кэша IE.

## Проверка на существующую реализацию в Android

- **Результат:** Функциональность реализована, но совершенно другим, специфичным для Android способом.
- **Объяснение**:
    1.  Код из `ExplorerHelper.cs` не может быть портирован, так как он полностью зависит от Windows API (`wininet.dll`).
    2.  В Android-приложении используется собственный механизм кэширования, реализованный в классе `ru.neverlands.abclient.proxy.Cache`.
    3.  В `SettingsActivity.java` есть пункт меню "Очистка кэша" (`clear_cache`).
    4.  Обработчик нажатия на этот пункт вызывает метод `ru.neverlands.abclient.proxy.Cache.clear()`, который и выполняет очистку кэша, используемого приложением.

## Решение для портирования на Android

Никаких действий не требуется. Задача очистки кэша уже решена в Android-проекте идиоматичным для этой платформы способом.
