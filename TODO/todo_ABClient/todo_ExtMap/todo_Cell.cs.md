
### 1. План портирования Cell.cs

Файл `Cell.cs` представляет собой класс-модель, описывающий одну ячейку (локацию) на игровой карте.

### 2. Функциональность в C#

- **Назначение:** Хранить всю известную информацию о конкретной локации на карте.
- **Свойства:**
    - `CellNumber`: Строковый идентификатор ячейки (например, "1-001").
    - `Name`: Название локации.
    - `HasFish`, `HasWater`, `HerbGroup`: Флаги и информация о ресурсах в локации.
    - `Updated`, `NameUpdated`, `CostUpdated`: Временные метки последнего обновления различных данных о ячейке.
    - `MapBots`: Список объектов `MapBot`, представляющих монстров в этой локации.
    - `MinBotLevel`, `MaxBotLevel`: Минимальный и максимальный уровень ботов в локации.
- **Особая логика:**
    - Свойства `Tooltip` (описание) и `Cost` (стоимость перехода) реализованы с фолбэком. Сначала они пытаются получить данные из глобального справочника `Map.AbcCells`. Если там нет информации для данной ячейки, они возвращают свои собственные, внутренние значения. Это говорит о наличии двух слоев данных: статического "атласа" и динамически обновляемой карты.
- **Методы:**
    - `AddMapBot()`: Добавляет бота в ячейку и обновляет агрегированную информацию о ботах.
    - `IsBot()`: Проверяет, есть ли в ячейке бот с заданным именем.
    - `NormalizeRegNum()`: Статический метод для форматирования идентификатора ячейки.

### 3. Решение для портирования на Android

Это ключевая модель данных для всего, что связано с картой. Класс должен быть портирован с сохранением всей его логики.

### 4. План реализации

1.  **Создать data-класс `Cell` в Kotlin:**
    - Перенести все простые свойства (`Name`, `HasFish` и т.д.).
    - `MapBots` будет `val mapBots: MutableList<MapBot> = mutableListOf()`.
    - `CellNumber` будет реализован как свойство с кастомными геттером и сеттером.
2.  **Реализовать логику с фолбэком:**
    - Свойства `tooltip` и `cost` должны будут обращаться к статическому или синглтон-объекту `MapManager`, чтобы получить данные из "атласа" (`AbcCells`).
