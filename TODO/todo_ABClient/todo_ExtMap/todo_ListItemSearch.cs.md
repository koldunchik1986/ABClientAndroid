
### 1. План портирования ListItemSearch.cs

Файл `ListItemSearch.cs` представляет собой структуру данных для одного элемента в списке результатов поиска по карте.

### 2. Функциональность в C#

- **Назначение:** Хранить информацию о найденной локации.
- **Поля:**
    - `RegNum`: Идентификатор найденной ячейки.
    - `Jumps`: Количество шагов (расстояние) до найденной ячейки.
    - `Html`: HTML-форматированная строка с подробной информацией о результате для отображения.
- **Методы:**
    - `ToString()`: Возвращает простую текстовую сводку, например, `"1-001 (шагов: 5)"`.

### 3. Решение для портирования на Android

Это тривиальный класс. Его следует портировать как data-класс в Kotlin или простой POJO в Java.

### 4. План реализации

1.  **Создать data-класс `ListItemSearch`:**
    - **Kotlin:**
      ```kotlin
      data class ListItemSearch(
          val regNum: String,
          val jumps: Int,
          val html: String // Или, что лучше, структурированные данные вместо HTML
      )
      ```
    - **Примечание:** Хранить `html` для отображения в нативном списке - плохая практика. В идеале, этот класс должен содержать структурированные данные, а UI-компонент (например, элемент `RecyclerView`) будет сам решать, как их отображать.
