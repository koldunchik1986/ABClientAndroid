# План портирования Dice.cs

Файл `Dice.cs` - это простой статический класс-обертка для генерации случайных чисел.

## Функциональность в C#

*   **Назначение**: Предоставить централизованный и простой доступ к генератору случайных чисел.
*   **Реализация**: 
    *   Создает один статический экземпляр `System.Random`.
    *   Предоставляет два перегруженных метода `Make` для генерации случайного числа либо до максимального значения, либо в заданном диапазоне.

## Проверка на существующую реализацию в Android

- **Результат:** Функциональность не реализована. В Android-проекте нет класса `Dice`.

## Решение для портирования на Android

Необходимо создать аналогичный статический класс `Dice.java`, используя `java.util.Random`.

## План реализации

- [ ] **Создать файл `Dice.java`** в пакете `ru.neverlands.abclient.utils`.

- [ ] **Реализовать статический класс**:
    ```java
    import java.util.Random;

    public final class Dice {
        private static final Random RAND = new Random();

        private Dice() {}

        public static int make(int max) {
            return RAND.nextInt(max);
        }

        public static int make(int min, int max) {
            // nextInt(max - min) вернет значение от 0 до (max - min - 1)
            // поэтому добавляем 1 к разнице, чтобы включить верхнюю границу
            return RAND.nextInt(max - min + 1) + min;
        }
    }
    ```
    *Примечание: Логика `make(min, max)` в Java `Random` немного отличается от C#, поэтому ее нужно скорректировать, чтобы она соответствовала поведению C# (включая `max` в возможный результат). Однако, стандартный `Rand.Next(min, max)` в C# НЕ включает `max`. Java `nextInt(bound)` также не включает `bound`. Поэтому прямой перенос `RAND.nextInt(max - min) + min` будет работать, если `max` не должен включаться. Судя по коду `AutoAnswerMachine`, где `j = Helpers.Dice.Make(prepAutoAnswers.Length)`, `max` не включается. Поэтому простая реализация `RAND.nextInt(max)` и `RAND.nextInt(max-min) + min` будет верной.*
