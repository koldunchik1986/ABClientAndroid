# План портирования TypeStat.cs

Файл `TypeStat.cs` определяет класс для хранения различной статистики, собранной во время игровой сессии.

## Функциональность в C#

*   **Назначение**: Служить контейнером для множества статистических данных: трафик, опыт, валюта, дроп и т.д.
*   **Свойства**:
    *   `LastReset`, `LastUpdateDay`, `Reset`: Управление сбросом статистики.
    *   `Traffic`, `SavedTraffic`: Статистика по сетевому трафику.
    *   `Drop`, `ItemDrop`: Информация о выпавших предметах.
    *   `XP`, `NV`, `FishNV`: Статистика по опыту и валюте.

## Проверка на существующую реализацию в Android

- **Результат:** Функциональность не реализована в виде единого класса.
- **Объяснение**: Некоторые поля из этого класса (например, `FishNV`) существуют как отдельные статические переменные в `AppVars.java`. Отсутствует единая структура для хранения всей статистики.

## Решение для портирования на Android

Необходимо создать POJO-класс `TypeStat.java` и включить его в `UserConfig.java`. Это позволит централизовать всю статистику в одном месте, что упростит управление ею.
