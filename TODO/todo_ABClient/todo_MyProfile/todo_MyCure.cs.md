# План портирования MyCure.cs

Файл `MyCure.cs` определяет класс для хранения всех настроек, связанных с функцией автоматического лечения.

## Функциональность в C#

*   **Назначение**: Инкапсулировать все настройки авто-лечения, включая цены, шаблоны сообщений и флаги активации, а также логику их сохранения и загрузки в/из XML.
*   **Свойства**: Хранит несколько массивов и полей:
    *   `NV`: Цены за лечение 4-х типов травм.
    *   `Ask`: Шаблоны сообщений с вопросом о лечении.
    *   `Adv`: Шаблон рекламного сообщения.
    *   `After`: Сообщение после успешного лечения.
    *   `Boi`: Сообщение, если цель в бою.
    *   `Enabled`: Флаги, разрешающие/запрещающие лечение каждого типа травм.
    *   `Disabled04`: Флаг, отключающий лечение для низкоуровневых персонажей.
*   **Сохранение и загрузка**: Методы `Write(XmlWriter)` и `Read(XmlReader)` отвечают за сериализацию и десериализацию. Примечательно, что сохраняются только те значения, которые отличаются от значений по умолчанию.

## Проверка на существующую реализацию в Android

- **Результат:** Функциональность не реализована.

## Решение для портирования на Android

Как и `MyAutoAnswer`, этот класс является частью профиля пользователя. Его следует портировать как POJO-класс `MyCure.java` и включить в качестве поля в `UserConfig.java`. Сериализация будет автоматически обработана `Gson`.

## План реализации

- [ ] **Создать файл `MyCure.java`** в пакете `ru.neverlands.abclient.model`.
    - [ ] Поля: `public int[] nv;`, `public String[] ask;`, `public String adv;`, `public String after;`, `public String boi;`, `public boolean[] enabled;`, `public boolean disabled04;`.
    - [ ] В конструкторе инициализировать все поля значениями по умолчанию, взятыми из C#-версии.

- [ ] **Интегрировать в `UserConfig.java`**:
    - [ ] Добавить новое поле: `public MyCure cure = new MyCure();`.
    - [ ] `Gson` автоматически обработает сохранение и загрузку. Так как в C# сохранялись только измененные поля, а `Gson` по умолчанию сохраняет все, поведение немного изменится, но это приемлемо и даже проще.
