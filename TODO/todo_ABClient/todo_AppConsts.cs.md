# План портирования AppConsts.cs

Файл `AppConsts.cs` является центральным хранилищем констант для всего C# приложения.

## Функциональность в C#

*   **Назначение**: Содержит сотни статичных `readonly` строковых и числовых констант, сгруппированных по категориям.
*   **Категории**: 
    *   Ошибки и локализация.
    *   Криптография (соли, пароли).
    *   Форматы строк.
    *   Файловая система (расширения, имена файлов, папки).
    *   Сетевые адреса и хосты.
    *   Константы для работы прокси (заголовки, статусы).
    *   HTML/XML теги и атрибуты для парсинга.
    *   Тексты для диалоговых окон.
    *   HTML-фрагменты для внедрения в страницы.

## Проверка на существующую реализацию в Android

- **Результат:** Частично реализовано. Существует файл `app/src/main/java/ru/neverlands/abclient/utils/AppConsts.java`, но он содержит лишь малую долю всех необходимых констант. Многие важные секции (прокси, большинство URL, форматы, теги) отсутствуют.

## Решение для портирования на Android

Необходимо дополнить существующий файл `AppConsts.java`, перенеся в него все недостающие константы из C#-версии. При переносе следует адаптировать константы под реалии Android.

## План реализации

- [ ] **Открыть `AppConsts.java` и `AppConsts.cs`** для сравнения.

- [ ] **Перенести недостающие константы**, секция за секцией:
    - [ ] **Форматы**: Дополнить все `...Format` строки.
    - [ ] **Файловая система**: Перенести константы для имен файлов и папок. Пути, возможно, потребуют адаптации под файловую систему Android (например, использование `Context.getCacheDir()`).
    - [ ] **Адреса и хосты**: Перенести все недостающие URL (`GameReferalUrl`, `GameCaptchaUrl` и т.д.).
    - [ ] **Прокси**: Полностью перенести секцию с константами для прокси. Они будут необходимы при портировании `ABProxy`.
    - [ ] **Теги**: Перенести все `Tag...` и `Attr...` константы. Они критически важны для парсинга ответов сервера.
    - [ ] **Сообщения**: Перенести все `Message...` строки. В Android они, возможно, будут заменены ссылками на строковые ресурсы (`R.string...`), но для начала их можно перенести как есть.
    - [ ] **HTML-фрагменты**: Перенести все `Html...` строки. Они понадобятся для модификации страниц в `WebView`.

- [ ] **Адаптация под Android**:
    - [ ] Заменить константы, определяющие теги для XML-конфигурации (`ConfigTableMain`, `ConfigTableContacts` и т.д.), на ключи для `SharedPreferences`, если это еще не сделано.
    - [ ] Проверить, что константы, связанные с путями к файлам, не содержат жестко заданных путей Windows.

- [ ] **Обновить `todo_ABClient.md`**, отметив `AppConsts.cs` как проанализированный.
