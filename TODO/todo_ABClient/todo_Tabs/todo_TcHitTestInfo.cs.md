
### 1. План портирования TcHitTestInfo.cs

Файл `TcHitTestInfo.cs` содержит структуру, используемую для `hit testing` - определения, на какую часть кастомного `TabControl` нажал пользователь.

### 2. Функциональность в C#

- **Назначение:** Передать координаты клика мыши в нативную функцию WinAPI и получить обратно флаги, описывающие, на какой части вкладки был сделан клик.
- **Структура:**
    - `pt`: `Point` (X, Y) с координатами клика.
    - `flags`: `TcHitTestFlags`, которые заполняются в результате вызова нативной функции.
- **`[StructLayout(LayoutKind.Sequential)]`**: Этот атрибут подтверждает, что структура используется для взаимодействия с нативным кодом (P/Invoke).

### 3. Решение для портирования на Android

Этот класс, как и `TcHitTestFlags.cs`, **полностью специфичен для Windows** и не портируется. В Android нет необходимости вручную определять координаты клика по элементам UI, таким как вкладки.

### 4. План реализации

- Никаких действий не требуется. Этот класс не портируется.

- [x] Файл не требует портирования.
