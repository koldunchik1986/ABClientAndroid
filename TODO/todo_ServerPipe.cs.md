# План портирования ServerPipe.cs

Файл `ServerPipe.cs` является низкоуровневым компонентом кастомного прокси-сервера. Это простая обертка над классом `Socket`, аналогичная `ClientPipe.cs`.

## Функциональность в C#

*   **Хранение сокета**: Класс инкапсулирует объект `Socket`, представляющий TCP-соединение между прокси-сервером и удаленным веб-сервером (например, `neverlands.ru`).
*   **Настройка сокета**: Устанавливает TCP-параметр `NoDelay = true` для этого соединения.

## Решение для портирования на Android

**Портировать этот класс не нужно.**

Этот класс полностью относится к ручному управлению TCP-сокетами. При использовании нативного для Android подхода с `WebViewClient` и `OkHttp`, мы не имеем и не нуждаемся в доступе к низкоуровневым сокетам. Вся работа с TCP-соединениями, их настройка и управление инкапсулированы внутри системного сетевого стека Android и используемой HTTP-библиотеки.

Таким образом, вся функциональность этого класса становится нерелевантной.
