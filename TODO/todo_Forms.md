# Общий анализ папки Forms

Папка `Forms`, как и `ABForms`, содержит набор окон (форм) пользовательского интерфейса, созданных на Windows Forms. Судя по названиям, эти формы отвечают за управление профилями, аутентификацию и некоторые другие вспомогательные функции.

## Содержимое папки

*   **`FormProfiles.cs`**: Ключевая форма в этой папке. Вероятнее всего, это менеджер профилей, который позволяет пользователю создавать новые профили, выбирать существующий для входа или удалять ненужные.
*   **`FormProfile.cs`**: Форма для редактирования настроек выбранного профиля (логин, пароль, настройки автоматизации и т.д.).
*   **`AskPassword.cs`, `NewPassword.cs`**: Простые диалоговые окна для ввода пароля (запрос существующего или создание нового).
*   **`AutoLogon.cs`**: Форма, связанная с процессом автоматического входа в игру.
*   **`HerbNavigator.cs`**: Форма "Навигатор по травам". Функционально отличается от остальных, представляет собой отдельный инструмент.

## Решение для портирования на Android

**Прямое портирование этих форм невозможно.**

Вся эта функциональность должна быть реализована с использованием нативных компонентов и подходов Android.

*   **Управление профилями**: Вместо отдельных окон, это может быть реализовано как один или несколько экранов (`Activity` или `Fragment`) в приложении. Например, стартовый экран со списком профилей (`RecyclerView`) и кнопками "Добавить"/"Редактировать"/"Удалить".
*   **Редактирование профиля**: Экран редактирования профиля в Android обычно реализуется с помощью `PreferenceScreen`, который предоставляет стандартный и удобный для пользователя интерфейс для изменения настроек.
*   **Диалоги**: Окна для ввода пароля должны быть заменены на `AlertDialog` с `EditText` внутри.
*   **Хранение данных**: Данные профилей (логины, пароли, настройки) должны храниться не в файлах, как это могло быть в десктопной версии, а в `SharedPreferences` или в базе данных `Room`.
