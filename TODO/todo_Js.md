# Общий анализ папки Js

Папка `Js` содержит различные версии JavaScript-файлов, которые используются как ресурсы или резервные копии.

## Содержимое папки

*   **`hp.js.org`**: Оригинальная (судя по расширению `.org`) версия скрипта `hp.js`.
*   **`map.js`**: Вероятно, кастомная версия скрипта карты, используемая клиентом.
*   **`map.js.abc`**: Еще одна версия скрипта карты.
*   **`map.js.org`**: Оригинальная версия скрипта `map.js`.

## Решение для портирования

Эти файлы являются **ассетами (ресурсами)**, а не кодом для портирования.

Действия с ними зависят от того, как они используются в логике C#-клиента, в частности, в `PostFilter`:

*   **Если фильтр заменяет серверный файл локальным**: Например, если `MapJs.cs` читает содержимое `ABClient/Js/map.js` и отдает его вместо ответа сервера, то нам нужно будет скопировать этот файл в `app/src/main/assets/js/` нашего Android-проекта и реализовать в `MapJs.java` чтение из `AssetManager`.
*   **Если фильтр только модифицирует скачанный файл**: Если фильтры `HpJs.cs` и `MapJs.cs` просто изменяют скрипты, полученные от сервера, то эти локальные файлы являются лишь резервными копиями для разработчика и **никаких действий с ними не требуется**.

## План действий

1.  Перепроверить реализацию `HpJs.cs` и `MapJs.cs`, чтобы окончательно определить, используются ли эти локальные файлы.
2.  Если используются — скопировать их в `assets` Android-проекта.
3.  Если не используются — проигнорировать эту папку.
