# План портирования WebBrowserExtendedNavigatingEventArgs.cs

Файл `WebBrowserExtendedNavigatingEventArgs.cs` — это простой класс для передачи данных, используемый в кастомных событиях `ExtendedWebBrowser`.

## Функциональность в C#

*   **Наследование**: Наследуется от `CancelEventArgs`, что добавляет ему булево свойство `Cancel`.
*   **Назначение**: Служит в качестве аргумента для событий `BeforeNavigate` и `BeforeNewWindow`.
*   **Свойства**: Хранит URL (`Address`), имя целевого фрейма (`Frame`) и флаг отмены (`Cancel`). Обработчик события может прочитать `Address` и `Frame`, и на основе этой информации установить `Cancel = true`, чтобы заблокировать навигацию.

## Решение для портирования на Android

**Портировать этот класс не нужно.**

Этот класс является частью внутренней реализации кастомной системы событий. В Android, при работе с `WebView`, мы используем стандартные колбэки и их аргументы, которые предоставляет система.

*   **Для `BeforeNavigate`**: Аналогом является метод `shouldOverrideUrlLoading(WebView view, WebResourceRequest request)`. Вся информация о запросе содержится в объекте `WebResourceRequest`. Вместо установки `Cancel = true`, мы просто возвращаем `true` из этого метода, чтобы сообщить `WebView`, что мы обработали URL и ему не нужно продолжать навигацию.

*   **Для `BeforeNewWindow`**: Аналогом является метод `onCreateWindow(..., Message resultMsg)`. Информация для создания нового окна передается через объект `Message`. Мы также возвращаем `true`, чтобы показать, что мы обработали событие.

Таким образом, потребность в кастомном классе для передачи аргументов отпадает, так как эту роль выполняют стандартные системные классы `WebResourceRequest` и `Message`.
