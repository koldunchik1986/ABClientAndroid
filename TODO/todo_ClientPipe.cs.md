# План портирования ClientPipe.cs

Файл `ClientPipe.cs` является низкоуровневым компонентом кастомного прокси-сервера. Это простая обертка над классом `Socket`.

## Функциональность в C#

*   **Хранение сокета**: Класс инкапсулирует объект `Socket`, представляющий TCP-соединение между прокси-сервером и клиентом (браузером).
*   **Предоставление информации**: Предоставляет удобный доступ к IP-адресу и порту клиента.
*   **Настройка сокета**: Устанавливает некоторые TCP-параметры для этого соединения, например, отключает алгоритм Нейгла (`NoDelay = true`) и устанавливает таймаут на чтение.

## Решение для портирования на Android

**Портировать этот класс не нужно.**

Этот класс полностью относится к ручному управлению TCP-сокетами в реализации прокси-сервера. При использовании нативного для Android подхода с `WebViewClient.shouldInterceptRequest()`, мы не имеем и не нуждаемся в доступе к низкоуровневым сокетам. Вся работа с TCP-соединениями, их настройка и управление инкапсулированы внутри системного сетевого стека Android, который используется компонентом `WebView`.

Таким образом, вся функциональность этого класса становится нерелевантной.
