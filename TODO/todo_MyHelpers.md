# Общий анализ папки MyHelpers

Папка `MyHelpers` содержит еще один набор статических классов-утилит, во многом дублирующий функциональность папки `Helpers`.

## Содержимое папки

*   **`HelperConverters.cs`**: Утилиты для преобразования типов.
*   **`HelperDice.cs`**: Генератор случайных чисел (дубликат `Helpers/Dice.cs`).
*   **`HelperErrors.cs`**: Генерация HTML-страниц с сообщениями об ошибках.
*   **`HelperExterns.cs`**: Объявления `DllImport` (дубликат `Helpers/NativeMethods.cs`).
*   **`HelperHttp.cs`**: HTTP-утилиты (дубликат `Helpers/Http.cs`).
*   **`HelperPacks.cs`**: Утилиты для сжатия/распаковки (дубликат `Helpers/Pack.cs`).
*   **`HelperStrings.cs`**: Очень важный класс с утилитами для работы со строками, в частности, `SubString`, который эмулирует поведение `Mid()` из VB.

## Решение для портирования на Android

**Сохранять дублирующую структуру не нужно.** При портировании необходимо провести рефакторинг и создать единый, логически организованный набор утилитных классов в пакете `ru.neverlands.abclient.utils`.

## План портирования

1.  **Объединение**: Для каждой пары дубликатов (например, `HelperDice.cs` и `Dice.cs`) нужно будет проанализировать оба, и если в них есть уникальные полезные методы, перенести их все в один соответствующий Java-класс (например, `Dice.java`).
2.  **`HelperStrings.java`**: `HelperStrings.cs` содержит важный метод `SubString`, который часто используется в коде для парсинга HTML. Его необходимо аккуратно портировать на Java, сохранив его поведение.
3.  **`HelperErrors.java`**: Методы для генерации HTML-страниц с ошибками можно перенести в утилитный класс, который будет использоваться для создания `WebResourceResponse` с ошибкой.
4.  **Игнорирование дубликатов**: Остальные дубликаты (`HelperHttp`, `HelperPacks`, `HelperExterns`) портировать не нужно, так как их функциональность либо не нужна, либо уже покрыта планами для папки `Helpers`.
