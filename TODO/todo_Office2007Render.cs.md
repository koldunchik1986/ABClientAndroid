# План портирования Office2007Render.cs и Office2007ColorTable.cs

Файлы `Office2007ColorTable.cs` и `Office2007Render.cs` представляют собой единый механизм для полной кастомной отрисовки UI-элементов в стиле Microsoft Office 2007.

## Функциональность в C#

*   **`Office2007ColorTable`**: Представляет собой статическую палитру, определяющую сотни цветов для каждого возможного состояния UI-элементов (кнопки, меню, панели инструментов): цвет градиента при наведении, цвет границы при нажатии и т.д.
*   **`Office2007Render`**: Это кастомный "рендер", который использует палитру `Office2007ColorTable` и, переопределяя все методы `OnRender...`, вручную, пиксель за пикселем, рисует каждый элемент управления с помощью низкоуровневых вызовов `System.Drawing` (GDI+). Он отвечает за градиенты, формы, тени, стрелки и т.д.

**Назначение:** Заменить стандартный внешний вид Windows Forms на кастомный, имитирующий популярный на тот момент стиль Office 2007.

## Решение для портирования на Android

**Портировать эти классы не нужно и категорически не рекомендуется.**

1.  **Визуальная несовместимость**: Стиль Office 2007 является устаревшим и предназначен для десктопных приложений. Внедрение такого стиля в мобильное Android-приложение сделает его чужеродным, неудобным и непонятным для пользователя.
2.  **Технологическая несовместимость**: Вся логика основана на `System.Drawing` (GDI+), аналогов которому в Android нет. Попытка воссоздать подобную ручную отрисовку на `Canvas` будет невероятно трудоемкой и неэффективной.

## План реализации (вместо портирования)

Внешний вид Android-приложения должен соответствовать гайдлайнам платформы.

1.  **Использовать Material Design**: Весь UI должен быть построен с использованием компонентов из библиотеки **Material Components for Android** (`com.google.android.material`). Это стандарт де-факто для современных Android-приложений.
    *   Вместо `ToolStrip` -> `com.google.android.material.appbar.MaterialToolbar`.
    *   Вместо `MenuStrip` и `ContextMenuStrip` -> `Menu` в XML, `PopupMenu`.
    *   Вместо `Button` -> `com.google.android.material.button.MaterialButton`.

2.  **Стилизация через темы**: Цветовая палитра, шрифты, формы углов и другие аспекты внешнего вида должны определяться в файлах тем и стилей (`res/values/themes.xml`, `res/values/colors.xml`, `res/values/styles.xml`).
    *   Это позволяет централизованно управлять внешним видом всего приложения.
    *   Обеспечивает простую поддержку системной темной/светлой темы.

Таким образом, вместо портирования устаревшего кастомного рендера, необходимо создавать UI с использованием современных, нативных для Android инструментов и принципов дизайна.
