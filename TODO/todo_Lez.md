# Общий анализ папки Lez

Папка `Lez` содержит набор классов, реализующих логику автоматического боя (автобота) для специфической игровой локации — предположительно, Ледяной пещеры ("Lez").

## Содержимое папки

*   **`FormSettingsAb.cs`**: UI-форма для детальной настройки параметров автобоя именно в этой локации.
*   **`LezBotsClass.cs`, `LezBotsClassCollection.cs`, `LezBotsGroup.cs`**: Набор data-классов для описания ботов, встречающихся в этой локации, их характеристик и группировки.
*   **`LezFight.cs`**: Основной класс, содержащий тактическую логику автобоя: выбор цели, порядок использования заклинаний, принятие решений в зависимости от ситуации в бою.
*   **`LezSpell.cs`, `LezSpellCollection.cs`**: Data-классы для описания заклинаний, которые бот может использовать.
*   **`LezNode.cs`**: Вероятно, класс, описывающий узел в дереве принятия решений или в другой структуре данных, используемой для алгоритма боя.

## Решение для портирования на Android

Это сложный, узкоспециализированный модуль, который является расширением основной системы автобоя.

**Статус: БЛОКИРОВАНО.** Портирование этого модуля должно происходить **после** полной реализации основной системы автобоя (см. `TODO/todo_FightJS.cs.md`).

## План портирования (на будущее)

1.  **Портирование структур данных**: Перенести на Java классы `LezBots...` и `LezSpell...` как простые POJO-классы.
2.  **Интеграция с `AutoboiManager`**: Логику из `LezFight.cs` не нужно создавать как отдельный модуль. Вместо этого, ее следует интегрировать в уже существующий `AutoboiManager` (который будет создан при портировании `FightJs`). Например, `AutoboiManager` может иметь метод `getTactics(location)`, который будет возвращать специфический тактический профиль в зависимости от текущей локации игрока. Для Ледяной пещеры он будет возвращать тактику, портированную из `LezFight.cs`.
3.  **Интеграция настроек**: Все настройки из `FormSettingsAb.cs` должны быть перенесены в общий экран настроек Android-приложения (на основе `PreferenceScreen`). Для них можно создать отдельную категорию "Автобой: Ледяная пещера".

Таким образом, этот модуль не будет существовать как отдельная сущность в Android-проекте, а его логика и настройки будут интегрированы в общую систему автобоя и настроек.
