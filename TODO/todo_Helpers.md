# Общий анализ папки Helpers

Папка `Helpers` содержит набор статических классов-утилит, предоставляющих вспомогательные функции, которые используются в различных частях приложения.

## Содержимое папки

*   **`Converters.cs`**: Содержит методы для преобразования данных из одного формата в другой (например, число минут в форматированную строку).
*   **`Crypts.cs`**: **Критически важный класс**. Реализует алгоритмы хэширования паролей и, вероятно, шифрования/дешифрования данных профиля. Его точное портирование необходимо для совместимости и безопасности.
*   **`Dice.cs`**: Простая обертка над генератором случайных чисел.
*   **`Http.cs`**: Вспомогательные методы для работы с HTTP.
*   **`Misc.cs`**: Различные мелкие утилиты, не вошедшие в другие классы.
*   **`NativeMethods.cs`**: **Непортируемый класс**. Содержит объявления `DllImport` для вызова функций из системных библиотек Windows (WinAPI). Любая функциональность, зависящая от этого класса, потребует поиска нативного Android-аналога.
*   **`Pack.cs`**: Утилиты для упаковки/распаковки данных (возможно, для сжатия или специфичного для игры формата).
*   **`Russian.cs`**: **Критически важный класс**. Предоставляет доступ к кодировке `windows-1251` (Codepage 1251), которая необходима для корректного чтения и отправки текста на игровой сервер.
*   **`SerialKey.cs`**: Содержит логику, связанную с проверкой лицензионного/серийного ключа приложения.

## План портирования

В отличие от UI-папок, большинство классов из `Helpers` **требуют прямого портирования** их логики на Java.

1.  **Создать пакет `utils`**: В Java-коде создать пакет `ru.neverlands.abclient.utils` для размещения портированных утилит.
2.  **Портировать классы**: Для каждого `.cs` файла создать соответствующий `.java` класс.
3.  **`Crypts.java` и `Russian.java`**: Уделить этим двум классам особое внимание. `Russian.java` должен корректно работать с `Charset.forName("windows-1251")`. `Crypts.java` должен использовать стандартные криптографические библиотеки Java (`java.security`, `javax.crypto`) для точного воссоздания алгоритмов хэширования и шифрования.
4.  **Заменить `NativeMethods.cs`**: Функциональность, которая зависела от этого класса, должна быть заменена вызовами Android SDK. Например, если там был вызов для отображения уведомления, в Android нужно использовать `NotificationManager`.
5.  **Адаптировать остальные утилиты**: Перенести код остальных утилит, заменяя типы данных и вызовы .NET Framework на их Java/Android аналоги.
